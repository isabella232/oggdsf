\section{Building against libannodex}
\label{group__building}\index{Building against libannodex@{Building against libannodex}}
\subsection{Using GNU autoconf}\label{autoconf}
If you are using GNU autoconf, you do not need to call pkg-config directly. Use the following macro to determine if libannodex is available:

\small\begin{alltt}
 PKG\_CHECK\_MODULES(ANNODEX, annodex $>$= 0.6.0,
                   HAVE\_ANNODEX="yes", HAVE\_ANNODEX="no")
 if test "x$HAVE\_ANNODEX" = "xyes" ; then
   AC\_SUBST(ANNODEX\_CFLAGS)
   AC\_SUBST(ANNODEX\_LIBS)
 fi
 \end{alltt}\normalsize 


If libannodex is found, HAVE\_\-ANNODEX will be set to \char`\"{}yes\char`\"{}, and the autoconf variables ANNODEX\_\-CFLAGS and ANNODEX\_\-LIBS will be set appropriately.\subsection{Determining compiler options with pkg-config}\label{pkg-config}
If you are not using GNU autoconf in your project, you can use the pkg-config tool directly to determine the correct compiler options.

\small\begin{alltt}
 ANNODEX\_CFLAGS=`pkg-config --cflags annodex`\end{alltt}\normalsize 


\small\begin{alltt} ANNODEX\_LIBS=`pkg-config --libs annodex`
 \end{alltt}\normalsize
